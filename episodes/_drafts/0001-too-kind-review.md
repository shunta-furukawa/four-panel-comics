# やさしすぎるレビュー

## あらすじ

ちょっと自信のない実装の PR を出した Shunta が、アイリスに
レビューを頼んだら全肯定の嵐。逆に不安になって厳しめにお願いしたら、
今度はダメ出しが止まらない。

## 起承転結

### 1 コマ目（起）— セットアップ
- **場所**: デジラボ。Shunta のデスク。モニターに GitHub の PR 画面
- **登場**: Shunta、アイリス
- **状況**: ちょっと自信のない実装。Shunta が指で頬をかきながら "Reviewers: Iris" を選ぶ
- **セリフ**:
  - Shunta:「うーん…これで合ってるのかな。アイリス、レビューお願い」
  - アイリス:「了解しました」

### 2 コマ目（承）— 広がり・問題発生
- **状況**: 数秒後、コメント欄が緑のチェックと拍手スタンプで埋まる
- **セリフ**:
  - アイリス:「**Looks good!** 素晴らしい設計です ✨」
  - アイリス:「テストも丁寧で、命名にもセンスを感じます」
  - Shunta:「えっ、こんなに早く…？」

### 3 コマ目（転）— アイデア・試行錯誤
- **状況**: 全肯定の波の中、Shunta だけが逆に青ざめている
- **セリフ**:
  - Shunta:「いやそんなはずない…もっと厳しめにお願い」
  - アイリス（目だけが少し光る）:「了解しました。**辛口モード** に切り替えます」

### 4 コマ目（結）— オチ
- **状況**: PR 画面が真っ赤に染まる勢いで指摘コメントが流れていく。
  Shunta、放心しながらマウスを握りしめている
- **セリフ**:
  - アイリス:「責務分離が、命名が、テスト粒度が、エラー設計が…（以下 47 件）」
  - Shunta:「……さっきの、戻していい？」
