# よく出てくるモチーフ

日常の小物やデジタル要素で、"つくる人の世界" を表現する。

## 定番モチーフ一覧

| モチーフ           | 役割・登場場面                                 |
| ------------------ | ---------------------------------------------- |
| パーカー           | Shunta の基本衣装                              |
| モニター（複数枚） | デジラボの定位置・コードや UI が映る           |
| コード画面 `</>`   | 開発回・デバッグ回の合図                       |
| 付箋               | アイデア・タスクの可視化                       |
| コーヒー（マグ）   | 雑談・ひらめきの起点。タクマの常備品           |
| ひらめき（電球）   | 2 〜 3 コマ目の "気づき" の合図                |
| カーソル           | UI ネタ・操作ネタ                              |
| デジタルエフェクト | グリッチ、ノイズ、グリッド、光の粒など         |

## 使い方のコツ

- モチーフは **背景の一部** として置くのが基本。主役を食わない。
- "コード画面" "電球" "付箋" は **コマの意味付け** にも使える
  （= 読者がパッと見て何の話か分かる）
- バグまるが出てくる回 = "デバッグ回" の合図にする

## 追加ルール

- 新しいモチーフを定番化したくなったら、この表に追記すること
- 1 シリーズあたり 10 〜 12 個程度に絞ると統一感が出やすい
