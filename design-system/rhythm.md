# 4 コマのリズム設計

> テンポよく読めて、最後にクスッと笑えるオチを目指す。

## 起承転結

| コマ | 役割    | 中身                              | NG パターン                              |
| ---- | ------- | --------------------------------- | ---------------------------------------- |
| 1    | **起**  | セットアップ・状況説明・導入      | 情報を詰め込みすぎる                     |
| 2    | **承**  | 広げる・問題の発生・気づき        | 起と同じ温度で進めてしまう               |
| 3    | **転**  | アイデア・試行錯誤・温度変化      | "笑い" を 3 コマ目で出してしまう         |
| 4    | **結**  | オチ：意外な結末・落ち or 発見    | 説明セリフでオチを台無しにする           |

## チェックリスト（投稿前）

- [ ] 1 コマ目で **誰が・どこで・何をしてるか** が分かるか
- [ ] 2 コマ目で **テーマや問題** がはっきりしているか
- [ ] 3 コマ目で **温度（テンション）が変化** しているか
- [ ] 4 コマ目で **オチが立っている**（説明していない）か
- [ ] セリフは音読してテンポが悪くないか
- [ ] バグまる回なら "デバッグ地獄" の絵的な合図が入っているか

## オチのバリエーション

- **意外な結末**: 想像と違う方向に終わる
- **発見**: 主人公が何かに気づく
- **空回り**: 結果として何も解決していないが面白い
- **無言オチ**: セリフなしで絵だけで落とす
- **メタオチ**: 4 コマというフォーマットや読者を意識した落とし方

## 参考: 作りやすいテンプレ

- "あるある提示 → 共感 → ひねり → 想定外のオチ"
- "問題発生 → 解決っぽいアイデア → 試す → 別の問題が発生"
- "ひらめき → 実装 → 動いた！ → バグまる出現"
