# 吹き出し / セリフのルール

> 漫画の絵作りで一貫性を保つための、吹き出しとセリフ周りの取り決め。
> 画像生成（ChatGPT など）にプロンプトを投げるときは、
> このルールを **共通仕様（初期セットアップ）に必ず含める**。

## なぜ明文化するか

ChatGPT を含む画像生成 AI は、プロンプトに `Shunta「セリフ」` のような
書き方をすると、`Shunta「」` ごと吹き出しに描いてしまう癖がある。
また、複数キャラがいるパネルで吹き出しの位置・テイル方向が
話者から外れることも多い。
これらは生成のクセで、ルール側で明示すれば安定して直せる。

## 吹き出しの基本ルール

- 各セリフの吹き出しは、必ず **話者の口元の近く** に配置する
- 吹き出しのテイル（尾の三角）は **話者の方向** を指す
- 2 人以上のキャラが同じパネルにいる場合:
  - **話者の側** に吹き出しを置く
  - 話者ではないキャラの方にテイルを向けない
  - 「誰が話しているか」が一目でわかる構図にする

## セリフの中身

- 吹き出しには **そのセリフ本文だけ** を入れる
- **話者名（Shunta / アイリス / タクマ など）を吹き出しの中に書かない**
- 連続したセリフを 1 つの吹き出しに詰め込みすぎない

## 話者の表示方法（キャラ名タグ）

- 各パネル左下に **話者名のタグ** を配置する
- スタイル: **ピンク背景 (#FF2D7A) + 白文字**、角丸の小さなタグ
- 1 パネルに複数の話者がいる場合、それぞれのタグを左下にまとめて並べる

## 画像生成プロンプトでの記述フォーマット

各パネルの仕様を書くときは、以下の形式に揃える:

```
# パネル N
- 場面: ...
- 話者: Shunta
- 吹き出し: ここに本文のみ
- キャラ名タグ: Shunta（左下、ピンク + 白）
```

NG（旧形式 / 使わない）:

```
- セリフ: Shunta「ここに本文」  ← Shunta「」 ごと吹き出しに描かれてしまう
```

## 共通仕様に貼り付けるテキスト（コピペ用）

画像生成プロンプトの「初期セットアップ」セクションに、以下を含める:

```
[吹き出しのルール]
- 各セリフの吹き出しは、必ず話者の口元の近くに配置する
- 吹き出しのテイル (尾の三角) は話者の方向を指す
- 2 人以上のキャラが同じパネルにいる場合、話者ではないキャラの方には
  吹き出しを置かない / テイルを向けない
- 吹き出しの中には「セリフ本文だけ」を入れる。話者名 (Shunta / アイリス /
  タクマ など) は吹き出しの中に書かない
- 話者名は各パネル左下に「ピンク背景 (#FF2D7A) + 白文字」のキャラ名タグ
  として表示する
- 1 パネルに複数の話者がいる場合は、それぞれのタグを左下にまとめて並べる
```

## チェックリスト（生成画像の確認時）

- [ ] 吹き出しが話者の口元近くにある
- [ ] テイルが話者の方向を指している
- [ ] 吹き出しの中に話者名が書かれていない
- [ ] 各パネル左下に話者名のキャラ名タグがある
- [ ] 2 人以上の場面で「誰が話しているか」が一目でわかる
